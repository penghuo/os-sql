# ClickBench Benchmark: Trino-OpenSearch vs Spark

**Date**: 2026-04-05
**Dataset**: [ClickBench hits](https://github.com/ClickHouse/ClickBench) — 99,997,497 rows, 105 columns, Iceberg table (Parquet + Zstd)

## Summary

Trino embedded in OpenSearch runs **42 of 43** ClickBench queries faster than Spark, completing the full suite in **82 seconds** — **2.5x faster** than Spark's 202 seconds.

| | Trino-OpenSearch | Spark |
|---|---|---|
| **Total time** | **82s** | 202s |
| **Avg per query** | **1.9s** | 4.7s |
| **Queries won** | **42** | 1 |
| **Fastest query** | 0.04s (Q0) | 0.78s (Q0) |
| **Slowest query** | 8.9s (Q23) | 39.4s (Q23) |

## Environments

| | Trino-OpenSearch | Spark |
|---|---|---|
| **Version** | Trino 440 in OpenSearch 3.6 | Spark (ClickBench reference) |
| **Machine** | 32 vCPU, 123 GB RAM | c6a.4xlarge (16 vCPU, 32 GB RAM) |
| **Heap / Memory** | 32 GB JVM | 32 GB RAM |
| **Table format** | Iceberg v2 (Parquet + Zstd) | Parquet |
| **Parallelism** | 4 in-process nodes, task_concurrency=32 | Default Spark config |
| **Methodology** | Best of 3 runs | Best of 3 runs |
| **Source** | This repo | [ClickBench c6a.4xlarge](https://github.com/ClickHouse/ClickBench/blob/main/spark/results/c6a.4xlarge.json) |

> **Hardware note:** Trino-OpenSearch uses 2x the vCPUs (32 vs 16). On equivalent 16-vCPU hardware, expect roughly 2x slower results, which would be on par with Spark.

## Q29: The Only Query Spark Wins

Q29 computes 90 `SUM(ResolutionWidth + N)` expressions over 100M rows. This is a vectorization microbenchmark:

- **Spark** reads the column in columnar batches and evaluates all 90 sums in tight vectorized loops
- **Trino** processes each row through 90 separate expression trees (Volcano iterator model)

The 1.19x gap (6.2s vs 5.23s) is inherent to the execution model, not a configuration issue.

## Key Optimizations

| Optimization | Impact |
|---|---|
| Iceberg table format (vs raw Parquet) | Metadata caching, predicate pushdown via column stats |
| 4-node in-process distributed execution | 4x parallelism for hash-partitioned GROUP BY / JOIN |
| `task_concurrency = 32` (was 4) | Match CPU count for pipeline parallelism |
| `dictionary_aggregation = true` | Faster GROUP BY on low-cardinality columns |
| Parquet read block: 64 MB / 65K rows | Larger batches reduce per-batch overhead |
| 32 GB heap, 70% for queries | No spill-to-disk needed |

## How to Reproduce

```bash
# 1. Download ClickBench data (14 GB, one-time)
wget -O trino/benchmark/data/hits.parquet \
  https://datasets.clickhouse.com/hits_compatible/hits.parquet

# 2. Create Iceberg table from Parquet via Spark
JAVA_HOME=/usr/lib/jvm/java-17-amazon-corretto.x86_64 \
  python3 trino/benchmark/create_iceberg_table.py /tmp/iceberg-clickbench-warehouse

# 3. Build plugin
./gradlew :opensearch-sql-plugin:bundlePlugin

# 4. Run OpenSearch with Trino enabled (32 GB heap)
./gradlew :opensearch-sql-plugin:run \
  -DenableTrino \
  -DtrinoHeap=32g \
  -DtrinoWarehouse=/tmp/iceberg-clickbench-warehouse

# 5. Create Iceberg table in Trino (CTAS from Hive)
#    See trino/benchmark/run_clickbench_iceberg.sh

# 6. Run benchmark
SKIP_START=1 OS_PORT=<port> TRIES=3 \
  ./trino/benchmark/run_clickbench_iceberg.sh
```
