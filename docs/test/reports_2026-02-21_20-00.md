### Analysis

**The 3 original serialization errors are fully resolved.** Plans now serialize and deserialize successfully through the Calcite JSON serde layer. However, two new categories of deserialization failures have been exposed:

1. **Plan root not Scannable (823 failures)**: After deserialization, `ShardCalciteRuntime.execute()` expects the plan root to implement `Scannable` (i.e., be a `DSLScan` or similar). Instead, the root is a standard Calcite node (`LogicalProject`, `LogicalSort`, `LogicalAggregate`, `LogicalSystemLimit`, `LogicalFilter`, or even `CalciteLogicalIndexScan` which deserializes but does not implement `Scannable`). This suggests the `PlanSplitter` is sending multi-node shard plans where the root is not the scan leaf, or the deserialized `CalciteLogicalIndexScan` loses its `Scannable` interface.

2. **UDF/custom function deserialization (921 failures)**: Plans containing user-defined functions (`UserDefinedFunctionBuilder$1`) or custom SQL operators (`SAFE_CAST`, `REGEXP_REPLACE`, `ILIKE`, `CONCAT`, etc.) fail during deserialization because:
   - Anonymous UDF classes lack default constructors required by Calcite JSON deser
   - Custom operators are not registered in the shard-side operator table

**Verdict**: All 3 original serialization issues are now resolved. The serde layer successfully serializes and deserializes plan JSON. However, the pass count is unchanged because two new categories of failures have been exposed:

1. **Plan root not Scannable (823)**: The deserialized shard plan's root node is not a `Scannable`/`DSLScan`. This is a plan structure issue -- either the `PlanSplitter` is including non-scan nodes at the root of shard plans, or the deserialized `CalciteLogicalIndexScan` does not implement `Scannable`.

2. **UDF/custom function deserialization (921)**: Anonymous UDF classes and custom SQL operators are not deserializable on the shard side. This requires either registering custom operators in the shard-side operator table or implementing custom serde for UDF nodes in `ExtendedRelJson`.

These are the next two issues to address to make progress on the integration test pass rate.
